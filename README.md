# Ori — 근원의 나로 돌아가는 길

> 외상 후 스트레스(PTSD)와 일상 우울감을 함께 들여다보는 심리 지지 챗봇.
> PCL-5 / IES-R 표준 척도를 자연스러운 대화로 녹이고, Gemini 2.5 Flash로 더 사람다운 응답을 제공합니다.

---

## 무엇이 들어 있나요

| 파일 | 역할 |
|---|---|
| `index.html` · `style.css` · `script.js` | 프론트엔드 (정적 웹) |
| `backend.py` | Python Flask 백엔드 (Gemini 응답 + 익명 데이터 수집) |
| `README.md` | 이 문서 |

### 핵심 기능

**대화형 자가진단**
- PCL-5 (20문항) + IES-R (22문항)을 6개 대화 토픽으로 묶어 자유 대화로 진행
- NLP가 정도부사·증상 키워드를 분석해 자동 채점
- 결과는 PCL-5 4군집(B/C/D/E) 레이더 차트로 시각화

**개인화 학습 (기기 내)**
- 사용자 어휘를 감정 카테고리별로 누적 → 봇이 미러링 응답
- 엔티티(사람·장소·사건) + 테마(직장 스트레스·수면 어려움 등) 자동 추출
- 다음 세션에 자연스러운 맥락 호출 (*"지난번에 말씀하신 ○○은 어떠세요?"*)

**자연스러운 대화**
- 4가지 행동(짧은 인정 / 공감 / 감정 라벨링 / 질문)을 가중치 기반으로 변주
- 응답 길이·타이핑 시간·침묵 시간이 감정 강도에 따라 조정
- Gemini 2.5 Flash 연결 시 진짜 LLM 응답, 미연결 시 규칙 기반 폴백

**위기 대응**
- 3단계 키워드 감지 (high / medium / trauma)
- Stanley-Brown 안전 계획 모달
- 1393 / 1577-0195 핫라인 연결, 트라우마센터 안내

**데이터 윤리**
- 모든 학습 데이터는 기본적으로 기기 안에서만 작동
- 익명 기여 모드는 별도 옵트인 (감정 카테고리·점수만, 대화 원문 제외)
- 사용자가 언제든 모든 데이터 삭제·내보내기 가능

## 빠른 시작

### 1단계 — 프론트엔드만 먼저 띄우기 (백엔드 없이)

가장 간단한 방법: `index.html`을 브라우저로 직접 열기. VS Code의 *Live Server* 확장이 있으면 더 편해요.

이 상태에서도 모든 기능이 작동합니다 — 자가진단, 학습, 메모리, 안전계획. 다만 봇 응답은 **규칙 기반**으로 작동해요. 채팅 헤더에 *"규칙 기반"* 뱃지가 표시됩니다.

### 2단계 — Gemini 2.5 Flash 무료 키 발급

1. https://aistudio.google.com/apikey 접속
2. Google 계정 로그인
3. **Create API key** 클릭 → 새 프로젝트로 키 생성
4. 키 복사 (한 번만 보여주니 안전한 곳에 저장)

**무료 한도 (2026년 5월 기준)**
- Gemini 2.5 Flash: 분당 10 요청, 일 500 요청, 분당 25만 토큰
- 신용카드 불필요
- 학기 발표·시연용으로 충분

> ⚠️ 무료 티어는 Google이 입력을 모델 학습에 사용할 수 있어요.
> **개인 식별 정보와 과거 대화 원문은 전송하지 않음**
> **현재 사용자 메시지는 Gemini 응답 생성을 위해 원문 그대로 전송됨**

### 3단계 — 백엔드 실행

```bash
# 의존성 설치
pip install flask flask-cors sqlalchemy google-generativeai
pip install scikit-learn numpy        # 자기 발전 사이클 (위험도 분류기) 사용 시

# API 키 환경변수로 설정
export GEMINI_API_KEY="여기에_발급받은_키_붙여넣기"

# (선택) 모델 변경 — 기본은 gemini-2.5-flash
export GEMINI_MODEL="gemini-2.5-flash"

# 실행
python backend.py
```

성공하면 콘솔에 이렇게 떠요:
```
[Ori] Gemini 활성화: gemini-2.5-flash
 * Running on http://0.0.0.0:5000
```

### 4단계 — 프론트와 백엔드 연결

`script.js`의 `API_BASE`는 **환경을 자동 판단**합니다:

- 로컬에서 `index.html`을 열거나 `localhost`로 접속 → `http://localhost:5000` 자동 사용
- 배포 환경(다른 도메인) → 같은 도메인의 백엔드를 사용 (또는 코드에서 별도 URL로 교체)

별도 설정 없이 백엔드를 띄우면 자동 연결됩니다. 채팅 헤더에 **"AI · Gemini"** 뱃지가 뜨면 성공이에요.

## 백엔드 엔드포인트

| 엔드포인트 | 메서드 | 설명 |
|---|---|---|
| `/v1/health` | GET | 서버·Gemini 활성화 여부 확인 |
| `/v1/respond` | POST | 사용자 메시지 → Gemini 응답 (위험도·군집 컨텍스트 반영) |
| `/v1/contribute` | POST | 익명 데이터 수집 (옵트인 사용자만, 만족도 평균 포함) |
| `/v1/contribute/forget` | POST | 익명 ID로 본인 기여분 삭제 요청 |
| `/v1/admin/aggregate` | GET | ML 학습용 집계 데이터 *(관리자 인증 추가 필요)* |
| `/v1/admin/insights` | GET | 집단 인사이트 캐시 조회 (발표·디버그용) |
| `/v1/admin/satisfaction` | GET | 만족도 평가 집계 조회 (보고서·발표용) |
| `/v1/admin/train` | POST | 위험도 분류기 학습 트리거 |
| `/v1/admin/classifier` | GET | 분류기 학습 상태 조회 |

### `/v1/respond` 안전장치 (다층 방어)

1. **사전 필터** — 사용자 메시지에 위기 키워드(*자살, 죽고 싶, 자해…*) 있으면 LLM 거치지 않고 즉시 안전 응답
2. **시스템 프롬프트** — Ori 페르소나·의료법 회피·이모지 금지 등 명시
3. **Gemini Safety Settings** — 자해·위험 카테고리 BLOCK_LOW_AND_ABOVE
4. **사후 필터** — 응답에 위험한 단어(*자살 방법, 약을 먹…*) 검출 시 차단
5. **폴백** — API 실패 시 `fallback: true` 반환 → 프론트가 규칙 기반으로 자동 전환

## 배포 시 체크리스트

```
□ HTTPS 강제 (개인정보 전송이므로 필수)
□ CORS 화이트리스트 — 우리 도메인만 허용
□ Rate limiting — IP당 분당 5회 정도
□ DB는 PostgreSQL 또는 동급 (SQLite는 개발용)
□ PIPA 제3자 제공 동의 절차 준수
□ IP·UA·Referer 등 부가 정보 로깅 금지
□ GEMINI_API_KEY는 환경변수로만 (코드 커밋 금지)
□ Gemini 무료 티어는 입력 학습 사용 — 사용자 약관에 명시
□ 위기 신호는 LLM 거치지 않고 사전 차단 유지
□ 의료법 회피 고지문 필수 ("의료 행위가 아닌 정서 지지 도구")
```

## 자기 발전 사이클 (Self-Improvement Loop)

Ori는 사용자가 늘수록 **세 가지 방식으로 자동으로 발전**합니다.

### A. 집단 인사이트 자동 학습
모인 익명 데이터에서 다음 패턴이 매시간 자동 추출되고, Gemini 시스템 프롬프트에 자동 주입됩니다:

- 시간대별 부정 감정 분포 → *"새벽 시간대 사용자에겐 더 부드럽게"*
- PCL-5 군집 평균 → *"전체에서 회피 영역이 가장 무거움"*
- 감정 카테고리 비중 → *"가장 자주 표현되는 감정에 깊이 공명"*
- 임상 기준선 초과 비율 → *"X% 가 위험군이므로 적극적 전문가 연계"*

확인: `GET /v1/admin/insights` 또는 `GET /v1/health`

### B. 위험도 분류기 학습 (scikit-learn)
- 피처: 시간대 분포(24차원) + 감정 카테고리 비중(5차원) = 29차원
- 라벨: PCL-5 ≥ 33 (임상 기준선)
- 모델: Logistic Regression vs Random Forest 교차검증, **recall 우선**
  *(기획서 원칙: 거짓 양성보다 거짓 음성이 더 위험)*
- `.pkl` 저장 → 부팅 시 자동 로드
- 매 응답 호출 시 추정 위험도가 시스템 프롬프트에 추가됨

학습 트리거 (관리자):
```bash
curl -X POST http://localhost:5000/v1/admin/train
```

상태 확인:
```bash
curl http://localhost:5000/v1/admin/classifier
```

> 주의: 분류기 학습엔 PCL-5 자가점검을 완료한 익명 기여 데이터가 최소 10건(위험군·비위험군 각 3건 이상) 필요합니다. 개발·시연 단계에선 표본이 부족할 수 있어요.

